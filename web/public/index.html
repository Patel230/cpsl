CPSL

Package tools, permissions, files, network access, and language adapters into explicit sandbox images. The same runtime contract can live in a CLI, server, app host, browser, or mobile shell.

Mini-OS capsules for agents: Docker-shaped flow, TOML manifest, Luau inside, Rust modules at the edges. Try the demo, then jump to the build file.

Build, list, run.

A capsule starts as TOML. Name the sandbox, pick modules, turn on Python, pin network domains, then use the CLI loop everyone already knows.

Build

$ cpsl build -t browser-agent -f cpsl-web.toml
built browser-agent

List

$ cpsl sandboxes
NAME           MODULES         SIZE    BUILT
browser-agent  fs, http, json  7.4 MB  2026-05-06

Run

$ cpsl run browser-agent --python -- 'print("hello from inside")'
hello from inside

How does it work?

CPSL is a Luau VM wrapped with explicit Rust capabilities. Different languages, one capsule boundary.

Luau VM

Luau comes from Roblox: small, embeddable, fast, and battle-tested by millions of creators and players.

Rust modules

Files, JSON, HTTP, and other host powers are opt-in capabilities.

Language shims

Shell, Python, and Lua/Luau all end up inside the same capsule.

Python-on-Luau can win sometimes

Smoke suite: 12/12 matched CPython output; 9/12 were faster after transpilation.

./run_tests.sh

What CPSL is not

Short version: it is a language-runtime sandbox, not a machine.

Docker/OCI

No daemon. No Linux filesystem. No container image.

CPython

No pip. No native packages. Practical scripts only.

Kernel isolation

Manifest-gated VM sandbox, not a jail or microVM.

Early and hackable.

CPSL started on May 4, 2026. The CLI, browser demo, manifests, modules, and language shims work. The open edges are still very real:

  • Repo modules github.com/org/cpsl-mod-tool
  • Prebuilt capsules Build once, pull everywhere.
  • CPSL hub Push/pull capsule images.
